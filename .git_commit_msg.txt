fix(docker): remove pnpm-workspace.yaml from backend

The pnpm-workspace.yaml in backend/ was incorrectly causing
pnpm 9 (used in Docker) to treat the directory as a workspace root.

Since no packages[] field was defined, pnpm 9 failed with:
"ERROR packages field missing or empty"

The backend is a standalone project, not a monorepo workspace.
The pnpm-workspace.yaml was only added as an attempt to configure
allowBuilds (pnpm 11 feature), which is unnecessary since we now
use pnpm@9 in Docker which runs build scripts without restrictions.
