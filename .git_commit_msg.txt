fix(tsconfig): disable declaration to fix TS2883 in Docker build

TS2883 occurs when declaration:true is set and TypeScript tries to emit
.d.ts files referencing internal pnpm symlink paths (.pnpm/...) that are
not portable across environments.

The backend is a Node.js application (not a library), so emitting
declaration files serves no purpose and causes cross-environment build
failures when package resolution paths differ.

- tsconfig.json: declaration false, declarationMap false
