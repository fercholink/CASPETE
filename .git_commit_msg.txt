fix(pnpm): add onlyBuiltDependencies to lockfile header

In pnpm 11, build script approval is stored in the pnpm-lock.yaml
under the top-level `onlyBuiltDependencies` key (not in package.json).
The package.json field is only used to pre-populate the interactive
approve-builds menu.

This is the equivalent of running `pnpm approve-builds` and selecting
all four packages in an interactive session.

Approved packages:
  - @prisma/engines (Prisma query engine binary)
  - bcrypt (native node addon)
  - esbuild (native binary for bundling)
  - prisma (CLI generate step)

Also restored --frozen-lockfile in backend/Dockerfile since the
lockfile now contains the correct approval metadata.
