fix(docker): pin pnpm@9 to bypass ERR_PNPM_IGNORED_BUILDS

pnpm 11 introduces mandatory interactive build script approval
(ERR_PNPM_IGNORED_BUILDS) that cannot be bypassed in Docker via
lockfile metadata or package.json config alone.

pnpm 9 does not have this restriction — it runs build scripts
(prisma, bcrypt, esbuild) automatically without approval.

pnpm 9 is fully compatible with Node.js 22 and handles the same
lockfile format.

Changes:
- backend/Dockerfile: corepack pnpm@latest -> npm install -g pnpm@9
- frontend/Dockerfile: corepack pnpm@latest -> npm install -g pnpm@9
- Both use --no-frozen-lockfile for cross-version compatibility
